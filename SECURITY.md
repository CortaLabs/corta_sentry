# Security policy

Report suspected vulnerabilities privately to `security@cortalabs.com`. Include the affected version, reproduction steps, impact, and any suggested mitigation. Do not open a public issue before maintainers acknowledge a coordinated disclosure timeline.

Only the latest tagged release receives security fixes. Operators remain responsible for authorization, allowlist accuracy, reverse-proxy TLS, filesystem access, backups, and local host security. Never scan a network without explicit authorization.
